## 1. Chosen Slice

**Slice name:** `resume_pack` (Phase 1-C)

**Why this slice is safe:**
- Single entry point (`_bundle_resume_pack`) with one nested helper (`_clean_text`).
- Strictly read-only: performs three `SELECT` queries via SQLAlchemy ORM and returns a computed dict.
- Zero side effects: no DB writes, no LLM calls, no background tasks, no file I/O.
- Zero coupling to `palace/bridge.py`: all queries are inline ORM calls using `SessionLocal()`.
- No imports from `backend.main` required if extracted; dependencies are limited to `SessionLocal`, ORM models (`SagaDigest`, `ArcSummary`, `ChapterSummary`), and `to_kst`.
- Exact current source block is 123 lines in 0.8 (`backend/main.py:3472-3594`), making it the smallest currently accepted Phase 1 candidate.

### Existing 2.0 proximity

The Go codebase already has a placeholder `GET /continuity-pack/{chat_session_id}` (`handleContinuityPack`) in `group_narrative.go`. The `resume-pack` concept is adjacent but distinct:
- `continuity-pack` is a broader narrative-state snapshot.
- `resume-pack` is the specific P-4c hierarchy summary used for long-gap re-entry.

**Decision:** Keep `resume-pack` as a separate route so the compare harness can shadow parity-test it independently. If the team prefers consolidation, the acceptance criteria below still apply to whichever route owns the payload.

## 5. Acceptance Criteria for Future Go/Python Parity Work

1. **Route exists and compiles.** Go route `GET /sessions/{chat_session_id}/resume-pack` is registered and returns HTTP 200.
2. **Field parity.** Response JSON contains exactly these top-level fields:
   - `pack_status` (`"ready"` or `"empty"`)
   - `trigger` (`"resume" | "idle" | "empty"`)
   - `sources_used` (`["saga","arc","chapter"]` etc.)
   - `layer_count` (integer)
   - `assembled_text` (string or `""`)
   - `saga` (object or `null`)
   - `arc` (object or `null`)
   - `chapter` (object or `null`)
   - `assembly_note` (string)
3. **Nested object parity.** `saga`, `arc`, and `chapter` objects contain the same field names as the Python payload (e.g., `id`, `from_turn`, `to_turn`, `era_label`, `resume_pack_text`, `saga_summary`, `arc_index`, `arc_name`, `arc_status`, `arc_resume_text`, `core_conflict`, `chapter_index`, `chapter_title`, `resume_text`, `summary_text`, `created_at`).
4. **`_clean_text` behavior match.** Text helper truncates to `max_len - 3` and appends `"..."`; whitespace is collapsed via `" ".join(str(raw or "").split()).strip()`.
5. **Trigger normalization match.** `"resume"` and `"manual_resume"` → `"resume"`; `"idle"` and `"idle_reentry"` → `"idle"`; `"empty"` and `"empty_input"` → `"empty"`; everything else → `null`/HTTP 204 or empty body.
6. **Ordering match.** SQL queries order by `to_turn DESC`, then `id DESC`, and take `.first()`.
7. **Null-safety match.** Missing optional fields on ORM rows must not panic; they should serialize as `null` or zero values consistently with Python.
8. **Compare harness evidence.** `internal/compare/harness.go` (or equivalent) produces a parity report showing this route’s JSON schema and sample output matching the Python reference within the allowed variance list.
