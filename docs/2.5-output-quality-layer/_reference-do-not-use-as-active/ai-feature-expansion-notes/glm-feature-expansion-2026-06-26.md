# 멀티에이전트 RP 출력 감독 시스템 — GLM 기능 발전 제안 (2.5 Output Quality Layer)

Date: 2026-06-26
Author: GLM (glm-5.2:cloud)
Status: feature expansion note, not authoritative spec

본 문서는 `multiagent-rp-supervisor-feature-plan.md`을 읽고, "기능적으로 더
발전시킬 수 있는 지점"과 "빠져 있거나 약한 역할 정의를 보강하는 아이디어"를
중심으로 작성한 제안이다. Archive Center / DB / 장기기억 의존 금지, 유저 입력
불변, 전체 재작성 회피, bounded patch 중심이라는 제한을 그대로 유지한다.

## 0. 기존 문서에 대한 짧은 평가

- **차별점(1)·Director(8)·트리아지(9)·모드(10)·MVP 분리(11)**는 거의 건드릴 게
  없다. 그대로 채택해도 좋다.
- **역할 정의(3~7)가 "점검 항목"은 잘 잡았지만 "무엇을 점검하지 *말아야*
  하는지(역할 밖 신호)"가 빠져 있다.** 이게 없으면 Director가 중재할 때
  "관할 위반 어노테이션"을 걸러낼 수 없다. 아래 13번 카드로 보강.
- **문체(6)·연출(7)이 "다듬기"에 머물러 있다.** RP/소설 출력 감독이라는 이름에
  걸맞으면 이 두 역할은 "교정"이 아니라 **"서사 효과 설계"** 쪽으로 한 단계 더
  올릴 수 있다. 아래 14, 15번.
- **killer 후보가 Continuity/No-Regression 두 개뿐**인데, RP 사용자가 강하게
  체감하는 지점이 하나 더 있다 — **"누가 말했는지/상황이 어디인지 헷갈리는
  순간"**. 아래 16번을 세 번째 killer 후보로 제안.

## 1. 추가 기능 카드

### 13. 관할 경계 선언 (Domain Boundary Manifest)
- **목적:** 각 Steward가 "내 관할 밖 신호는 어노테이션하지 않는다"는 규칙을
  각 역할 프롬프트에 명시적으로 박아, Director의 중재 전에 노이즈를 줄인다.
- **사용자 체감 효과:** 충돌이 적어지고, Director 호출이 가벼워진다.
- **동작 흐름:** 역할 프롬프트에 `관할 = {...}, 비관할 = {...}` 명시 → 비관할
  신호는 "flag-only(사유만)"로 반환, 패치 제안 금지.
- **호출되는 에이전트:** 전 Steward(프롬프트 규약).
- **필요한 LLM 호출 수:** +0(규약만).
- **streaming 사용 시 동작:** 무관.
- **UI 설정:** 없음(내부).
- **구현 난이도:** 하.
- **MVP / 추후 / 실험:** **MVP**(역할 분리가 동작하려면 전제).
- **기존 플러그인과의 차별점:** 일반 멀티에이전트는 "다 보고 투표"지만, 여기는
  **관할이 코드 수준에서 분리**된다.

### 14. 서사 효과 설계자 (Prose Steward 고도화 — "Show-don't-Tell + 감각 비트")
- **목적:** 문체 역할을 "교정"에서 **"서사 효과 설계"**로 올린다. 핵심 순간의
  tell("그는 슬펐다")을 show(감각/행동 비트)로 전환 제안, 사소한 장면의 과잉
  연출은 쳐낸다.
- **사용자 체감 효과:** "AI체"라고 느껴지는 지점(감정을 직접 서술)이 줄고,
  중요 장면만 입체적.
- **동작 흐름:** 감정어 밀도 + 행동 비트 밀도 측정 → 핵심 순간 tell 탐지 →
  동일 구간 내 show 비트로 **치환(분량 동결)** 제안.
- **호출되는 에이전트:** 문체 Steward(고도).
- **필요한 LLM 호출 수:** 1(문체 호출 시).
- **streaming 사용 시 동작:** 완료 후.
- **UI 설정:** "show-don't-tell 적극도" 슬라이더.
- **구현 난이도:** 중상.
- **MVP / 추후 / 실험:** **추후**(MVP는 안전 다듬기만).
- **기존 플러그인과의 차별점:** 문법 교정기가 아니라 **서사 기법 단위의 편집**.

### 15. 페이싱/긴장 곡선 시각화 (Pacing Curve)
- **목적:** 응답 내 긴장/감정 세기를 구간별로 측정해 곡선을 그려주고,
  클라이맥스 조기 소진·김 빠지는 마무리를 시각적으로 드러낸다.
- **사용자 체감 효과:** "이 장면 왜 갑자기 끝났지"를 한눈에. Audit-only에서도
  가치.
- **동작 흐름:** 감정어/행동 강도 추정 → 구간별 스코어 → 곡선 + 급변 구간
  하이라이트.
- **호출되는 에이전트:** 연출 Steward(경량) 또는 무LLM 추정.
- **필요한 LLM 호출 수:** 0~1.
- **streaming 사용 시 동작:** 완료 후(비파괴).
- **UI 설정:** 곡선 표시 on/off.
- **구현 난이도:** 중.
- **MVP / 추후 / 실험:** **추후**(Audit-only 보너스).
- **기존 플러그인과의 차별점:** 텍스트만 보여주는 게 아니라 **서사 호흡을
  시각화**.

### 16. (3번째 killer 후보) 장면 정합 가드 — Scene Anchor Guard
- **목적:** "누가 이 대사를 했는지", "지금 어디인지", "몇 명이 있는지"가
  헷갈리는 순간을 잡는다. RP 몰입 파괴 빈도 3위.
- **사용자 체감 효과:** "이 대사 누구임?", "갑자기 장소 바뀜?"이 사라진다.
- **동작 흐름:** 직전 턴의 참가자/위치/시간을 anchor로 추출 → 초안의 대사
  귀속·공간·인원 수와 대조 → 불일치 구간만 패치 또는 플래그.
- **호출되는 에이전트:** 연출 + 세계관 혼합(한 호출로 anchor 대조).
- **필요한 LLM 호출 수:** 1 + Director.
- **streaming 사용 시 동작:** 완료 후.
- **UI 설정:** 민감도.
- **구현 난이도:** 중상.
- **MVP / 추후 / 실험:** **MVP 후반~추후**(세계관 역할과 묶어서).
- **기존 플러그인과의 차별점:** 단순 교정기는 "문장"만 본다. 이건 **장면의
  물리적 정합성**을 지킨다.

### 17. 대사 귀속 해결기 (Dialogue Attribution Resolver)
- **목적:** 3인 이상 대화에서 화자가 불분명한 대사를 검출하고, 문맥으로 귀속을
  제안(또는 플래그).
- **사용자 체감 효과:** "이 말 누가 했지?" 도중 멈춤 사라짐.
- **동작 흐름:** 대사 추출 → 직전 화자/호칭 단서 매칭 → 불분명 대사에 화자
  힌트 삽입(짧은 행동 비트 추가) 또는 플래그.
- **호출되는 에이전트:** 캐릭터 Steward 확장.
- **필요한 LLM 호출 수:** +0(캐릭터 호출 시 함께).
- **streaming 사용 시 동작:** 완료 후.
- **UI 설정:** 자동 삽입 vs 플래그만.
- **구현 난이도:** 중.
- **MVP / 추후 / 실험:** **추후**(캐릭터 역할 안정화 후).
- **기존 플러그인과의 차별점:** 화자 혼동은 RP 특유 문제인데 교정기는 안 본다.

### 18. 캐릭터 음성 지문 캐시 (Voice Fingerprint Cache)
- **목적:** 세션 단위(영구 저장 아님)로 각 캐릭터의 말투 지문(어미·호칭·문장
  길이·금기어)을 1회 추출해 캐싱, 이후 턴의 캐릭터 일관성 점검을 **LLM 없이**
  값싸게 수행.
- **사용자 체감 효과:** 캐릭터 말투 흔들림이 매 턴 잡히면서도 비용 증가 거의 0.
- **동작 흐름:** 첫 등장 턴에 지문 추출(1회) → 이후 턴은 규칙 비교만 → 이탈 시
  캐릭터 Steward 호출.
- **호출되는 에이전트:** 캐릭터(최초 1회) + 이후 무LLM.
- **필요한 LLM 호출 수:** 캐릭터당 1(세션) + 0(이후 턴).
- **streaming 사용 시 동작:** 무관.
- **UI 설정:** 지문 자동 학습 on/off.
- **구현 난이도:** 중.
- **MVP / 추후 / 실험:** **추후**(단, 캐릭터 역할 비용을 크게 깎는 열쇠).
- **기존 플러그인과의 차별점:** 매번 LLM으로 말투 검사하는 멀티에이전트와 달리
  **세션 캐시로 비용을 0에 수렴**.

### 19. 유저 제시 긴장 회수 추적 (Promise Hook Tracker)
- **목적:** 유저 입력이 던진 질문/긴장/도전을 응답이 회수했는지 대조. 회수 못
  하면 마무리에 훅/여운 보강 제안.
- **사용자 체감 효과:** "내가 던진 걸 캐릭터가 무시했네"가 줄고, 턴 마무리가
  다음 행동을 당김.
- **동작 흐름:** 유저 입력에서 "약속(질문/도전/긴장)" 추출 → 응답이 회수했는지
  대조 → 미회수 시 마무리 구간에 짧은 훅 삽입 제안.
- **호출되는 에이전트:** 플롯 Steward 확장.
- **필요한 LLM 호출 수:** +0(플롯 호출 시).
- **streaming 사용 시 동작:** 완료 후.
- **UI 설정:** 자동 훅 보강 vs 플래그만.
- **구현 난이도:** 중.
- **MVP / 추후 / 실험:** **추후**(Continuity Guard 안정화 뒤 자연 확장).
- **기존 플러그인과의 차별점:** 유저 *입력*은 안 건드리면서, 입력이 만든
  **서사 약속**을 응답이 지키게 한다.

### 20. 역할별 회신 스키마 강제 (Structured Finding Schema)
- **목적:** 모든 Steward가 `{구간, 유형, 심각도, 제안패치, 관할태그, 신뢰도}`만
  반환하도록 스키마를 강제해 Director 병합을 결정론화.
- **사용자 체감 효과:** 간헐적 실패·빈 응답·잘못된 포맷으로 인한 "감독 안 됨"
  감소.
- **동작 흐름:** 역할 프롬프트에 JSON 스키마 + 실패 시 빈 배열 반환 규약.
- **호출되는 에이전트:** 전 Steward.
- **필요한 LLM 호출 수:** +0.
- **streaming 사용 시 동작:** 무관.
- **UI 설정:** 없음.
- **구현 난이도:** 하~중.
- **MVP / 추후 / 실험:** **MVP**(2.1 프로토콜의 실행 세부).
- **기존 플러그인과의 차별점:** 자유 텍스트 비평이 아니라 **기계적으로 병합
  가능한 구조화 결과**.

## 3. killer 포지셔닝 제안

기존 2개에 **Scene Anchor Guard(16)**를 더해 3종 세트로:

- **Continuity Guard** — "내 행동/직전 사건이 무시되는 순간" 제거 (서사 신뢰).
- **Scene Anchor Guard** — "누가 말했는지/어디인지 헷갈리는 순간" 제거 (장면
  정합).
- **No-Regression Promise** — "켜두고 잊어도 손해 없음" (신뢰 기반).

셋 다 "RP에서 김 빠지는 순간"을 정면으로 겨냥하고, 셋 모두 bounded patch +
fail-safe 위에서 동작한다는 점에서 **"안전하게 켜두는 RP 감독기"**라는
포지셔닝이 더 또렷해진다.
