# 멀티에이전트 RP 출력 감독 시스템 — 기능 기획안

> 작성: Kimi (GitHub Copilot), 2026-06-26
> 
> 제약: Archive Center, DB, 장기기억 시스템에 의존하지 않음. 사용자 입력문은 수정하지 않음. 전체 응답 무조건 재작성 금지.

---

## 1. 기존 플러그인과의 차별점

| 기존 방식 | 이 플러그인의 차별점 |
|-----------|----------------------|
| 단순 후처리: 문법 교정, 반복 제거 | **역할별 전문 AI가 RP/소설의 서사적 품질**을 점검 |
| agents 플러그인: 여러 LLM이 각자 재작성 후 합침 | **bounded patch**: 전체 재작성 없이 문제 구간만 제한 수정 |
| 전체 출력 교체형 | **보호 마커(envelope)를 분리**하고 본문만 개선 |
| 단일 기준 검사 | **캐릭터/세계관/플롯/문체/연출** 5축 동시 검사 |
| 사용자 입력 개선 | **사용자 원문은 절대 수정**, 메인 모델 출력만 감독 |
| 상태 없음 | **턴 단위 감사 로그**로 즉시 피드백 가능 |

**핵심 차별점**: "더 나은 글"이 아니라 "설정된 세계관과 캐릭터 안에서 더 일관되고 감정적으로 설득력 있는 RP/소설"을 만드는 감독 시스템.

## 2. MDASH 스타일을 살리려면 필요한 기능

1. **분리된 전문 에이전트**: 한 개의 "좋은 글쟁이"가 아니라, 각자 다른 기준을 들고 검사.
2. **계약 기반 점검**: 각 에이전트가 JSON 형태의 `issues` + `patch suggestions` 반환.
3. **bounded patch engine**: 제안된 patch를 원문의 가변 영역에만 적용.
4. **보호 영역 분리**: 이미지 태그, 시스템 태그, 코드블록, 특수 마크업은 untouched.
5. **conflict arbiter**: 여러 에이전트 의견 충돌 시 우선순위와 병합 규칙.
6. **adaptive agent router**: 상황에 따라 필요한 에이전트만 호출.
7. **streaming-aware audit**: 스트리밍 중에는 audit-only, 완성 후 patch.
8. **user-facing trace**: 어떤 에이전트가 무엇을 지적했는지 UI로 노출.

## 3~7. 각 역할 AI의 점검 범위

### 3. 캐릭터 담당 AI (Character Auditor)

- 말투/어휘 일관성 (성격, 연령, 배경)
- 캐릭터가 이전 턴과 모순되는 행동/감정을 보이는지
- 캐릭터가 자신의 동기나 가치관에 맞게 반응하는지
- 다른 캐릭터와의 관계 설정 위반
- OOC(Out of Character) 징후

### 4. 세계관 담당 AI (World Auditor)

- 설정된 물리법칙/사회구조/역사 위반
- 시간/장소/계절/날씨 모순
- 종족/직업/마법체계 규칙 위반
- 등장 물건, 장소, 조직의 이름/속성 일관성
- "이 세계에 없는 개념"이 등장했는지

### 5. 플롯/서사 담당 AI (Plot Auditor)

- 전개가 이전 턴의 약속/위임/질문을 무시하지 않는지
- 새로운 사건이 갑작스럽게 끼어들지 않는지 (deus ex machina)
- 서사적 인과관계가 끊기지 않는지
- 사용자 입력의 의도가 반영되었는지
- 다음 턴으로 자연스럽게 이어지는지

### 6. 문체/문장 담당 AI (Style Auditor)

- 시점(1인칭/3인칭) 일관성
- 문장 리듬, 반복, 어색한 표현
- RP 스타일(행동 중심, 대화 중심, 서술 중심) 유지
- 시대/장르에 맞는 어휘 선택
- 지나친 설명(exposition dump) 감지

### 7. 연출/감정선 담당 AI (Direction Auditor)

- 감정의 고조/반전/잔여가 자연스러운지
- 장면의 긴장도/호흡이 적절한지
- 묘사가 시각적/청각적/촉각적 감각을 적절히 사용하는지
- "보여주기 vs 말하기" 비율
- 클리프행거/전환/암시 등 연출 기법의 적절성

## 8. 충돌 시 최종 결정자 (Conflict Arbiter)

**계층적 우선순위**:

1. **Safety veto**: 보호 마커 손상, 전체 재작성 시도, 사용자 입력 변경 시도 → 무조건 reject.
2. **Character veto**: 캐릭터 일관성 위반은 World/Plot보다 우선.
3. **World veto**: 세계관 하드 규칙 위반은 Style/Direction보다 우선.
4. **Plot veto**: 서사적 인과관계 파괴는 Style보다 우선.
5. **Style/Direction**: 서로 충돌 시 사용자가 선택한 우선 모드에 따름.
6. **Confidence threshold**: 각 에이전트의 confidence가 0.7 미만인 제안은 폐기.

**병합 규칙**:
- 동일 구간에 충돌 patch가 있으면 우선순위 높은 것만 적용.
- 충돌이 해결 불가능하면 **원문 유지** (fail-open).
- 사용자가 "감정선 우선" / "문체 우선" / "설정 우선" 모드를 선택하면 해당 축의 제안이 우선.

## 9. 비용/속도 최적화: 언제 일부 에이전트만 호출할까

| 조건 | 호출 에이전트 | 이유 |
|------|---------------|------|
| 짧은 응답 (< 200자) | Character + Style only | 플롯/세계관 검사할 여지가 적음 |
| 대화가 대부분인 턴 | Character + Direction | 문체/세계관은 최소화 |
| 장면 전환/전투/사건 | World + Plot + Direction | 설정 위반과 서사 연속성이 중요 |
| 이미지 생성 후 텍스트 | Style + Direction | 캐릭터/세계관은 이미지 태그로 대체 |
| 스트리밍 모드 | Audit-only 1개 에이전트 | 완성 후에만 전체 패널 |
| 사용자가 "빠른 모드" 선택 | Character + Style only | 2개로 제한 |
| 사용자가 "감독관 모드" 선택 | 전체 5개 | 품질 우선 |

**추가 최적화**:
- 이전 턴에서 문제 없었던 축은 skip.
- 동일한 캐릭터/장소가 등장하면 캐릭터/세계관 에이전트만 호출.
- LLM 호출을 병렬로 실행.

## 10. 사용자가 UI에서 선택할 수 있어야 하는 모드

| 모드 | 설명 |
|------|------|
| **Off** | 출력 감독 비활성화 |
| **Audit Only** | 문제점만 표시, 패치는 하지 않음 |
| **Light Patch** | 제한된 구간만 수정 |
| **Director Mode** | 전체 에이전트 패널 가동 |
| **Fast Mode** | Character + Style만, 빠른 호출 |
| **Lore-Heavy Mode** | World + Plot 우선 |
| **Emotion-First Mode** | Direction 우선 |
| **Streaming-Aware** | 완성 후에만 감독, 중간에는 audit-only |
| **Per-Genre Preset** | 판타지/현대/공포/일상 등 프리셋 |

## 11. MVP vs 추후 버전

### MVP에 꼭 넣을 것

1. **Character Auditor + Style Auditor**: 가장 직접적인 체감.
2. **Protected Envelope Guardian**: 이미지/시스템 태그 보호.
3. **Bounded Patch Engine**: 제한적 수정.
4. **Audit Only / Light Patch 모드**.
5. **Fail-open**: 실패 시 원문 유지.
6. **Trace UI**: 어떤 지적이 있었는지 표시.

### 추후 버전

1. World Auditor, Plot Auditor, Direction Auditor.
2. Multi-agent conflict arbiter.
3. Adaptive agent router.
4. Streaming-aware full pipeline.
5. Per-genre presets.
6. User preference learning (session-only).

## 12. 킬러 기능 제안

### 킬러 기능 1: "캐릭터 보이스 핑거프린트"
- 캐릭터의 말투, 습관어, 반응 패턴을 턴 단위로 추출해 일관성을 수치화.
- "이 캐릭터가 87% 자기다" 같은 점수 제공.

### 킬러 기능 2: "세계관 표면 검증기"
- 사용자가 설정한 간단한 규칙 목록(텍스트)을 입력받아 위반 여부 검사.
- DB 없이도 동작.

### 킬러 기능 3: "감정선 리허설"
- 현재 응답의 감정 곡선을 시각화.
- "긴장도가 너무 일정함", "반전이 너무 갑작스러움" 등 피드백.

### 킬러 기능 4: "장면 블로킹 어시스턴트"
- 등장인물의 위치, 시선, 동작 흐름이 모순되는지 검사.
- "A가 방금 앉았는데 다시 서 있음" 같은 물리적 모순 감지.

## 기능 아이디어 상세

### 1. 캐릭터 보이스 핑거프린트 감사 (Character Voice Fingerprint Auditor)

- **목적**: 현재 응답이 캐릭터의 고유 말투와 행동 패턴을 유지했는지 점검.
- **사용자 체감 효과**: "이 캐릭터가 갑자기 말투가 변했다"는 불만을 줄임. 캐릭터가 "더 자기다워짐".
- **동작 흐름**:
  1. 최근 3~5턴의 해당 캐릭터 대사/행동을 추출.
  2. LLM이 "보이스 핑거프린트"(습관어, 어휘 레벨, 반응 패턴, 금지 표현)를 요약.
  3. 현재 응답과 비교해 위반 항목과 bounded patch 제안.
- **호출되는 에이전트**: Character Auditor 1회.
- **필요한 LLM 호출 수**: 1회 (핑거프린트는 캐싱 가능, 이후 0회).
- **streaming 사용 시 동작**: 완성 후에만 호출. 스트리밍 중에는 audit-only로 표시.
- **UI 설정**: "캐릭터 일관성 강도 (약/중/강)", "보이스 핑거프린트 자동 추출 on/off".
- **구현 난이도**: 중.
- **MVP / 추후 버전 / 실험 기능**: MVP.
- **기존 플러그인과의 차별점**: 단순히 "말투가 이상함"이 아니라, **캐릭터별 음성 프로필을 동적으로 만들어 일관성을 수치화**.

---

### 2. 세계관 표면 규칙 검증기 (World Surface Rule Validator)

- **목적**: 사용자가 입력한 간단한 세계관 규칙(텍스트 목록)을 기준으로 출력 위반 검사.
- **사용자 체감 효과**: "이 세계에는 마법이 없는데 갑자기 마법이 나왔다" 같은 설정 붕괴 방지.
- **동작 흐름**:
  1. 사용자가 UI에 규칙 목록 입력 (예: "이 세계에는 마법이 없다", "모든 드래곤은 날개가 2쌍").
  2. LLM이 출력을 규칙과 비교.
  3. 위반 시 bounded patch 제안 또는 reject.
- **호출되는 에이전트**: World Auditor 1회.
- **필요한 LLM 호출 수**: 1회.
- **streaming 사용 시 동작**: 완성 후 호출. 규칙 위반 시 사용자에게 알림.
- **UI 설정**: "세계관 규칙 목록" 텍스트 에어리어, "위반 시 원문 유지 / patch 시도".
- **구현 난이도**: 하~중.
- **MVP / 추후 버전 / 실험 기능**: MVP (기본 규칙 검증).
- **기존 플러그인과의 차별점**: **외부 DB 없이 사용자가 직접 입력한 규칙만으로 즉시 검증**.

---

### 3. 플롯 약속 추적기 (Plot Promise Tracker)

- **목적**: 이전 턴에서 던진 질문, 위임, 암시, 위협이 현재 응답에서 무시되지 않는지 확인.
- **사용자 체감 효과**: "아까 물어본 건 대답 안 하고 넘어갔다"는 느낌 방지.
- **동작 흐름**:
  1. 최근 3턴에서 "열린 상태"인 요소(질문, 위협, 제안, 미스터리)를 추출.
  2. 현재 응답이 이를 회피/무시/해결했는지 판단.
  3. 회피 시 patch 제안.
- **호출되는 에이전트**: Plot Auditor 1회.
- **필요한 LLM 호출 수**: 1회.
- **streaming 사용 시 동작**: 완성 후 호출.
- **UI 설정**: "약속 추적 강도", "무시 허용 턴 수".
- **구현 난이도**: 중.
- **MVP / 추후 버전 / 실험 기능**: 추후 버전.
- **기존 플러그인과의 차별점**: **문법이 아니라 서사적 인과관계를 추적**.

---

### 4. 문체 레지스터 조율관 (Style Register Harmonizer)

- **목적**: 1인칭/3인칭, 현재/과거, 격식체/비격식체, 서술/대화 비율을 유지.
- **사용자 체감 효과**: "갑자기 시점이 바뀌었다", "설명이 너무 많아졌다" 방지.
- **동작 흐름**:
  1. 최근 응답에서 문체 레지스터를 추출.
  2. 현재 응답이 레지스터를 얼마나 유지했는지 점수화.
  3. 편차 큰 구간 patch 제안.
- **호출되는 에이전트**: Style Auditor 1회.
- **필요한 LLM 호출 수**: 1회.
- **streaming 사용 시 동작**: 완성 후 호출.
- **UI 설정**: "문체 일관성 강도", "대화 비율 유지".
- **구현 난이도**: 하.
- **MVP / 추후 버전 / 실험 기능**: MVP.
- **기존 플러그인과의 차별점**: **단순 문법 교정이 아니라 시점/어조/서술 비율 같은 RP 핵심 문체를 유지**.

---

### 5. 감정선 연출 감독 (Emotional Arc Director)

- **목적**: 응답의 감정 고조, 반전, 잔여가 자연스러운지 점검.
- **사용자 체감 효과**: "갑자기 감정이 확 바뀌었다"는 위화감 줄임.
- **동작 흐름**:
  1. 응답을 감정 단위로 분할.
  2. 각 단위의 감정 레이블과 강도 추출.
  3. 급격한 전환, 감정 누락, 과도한 반복 감지 후 patch 제안.
- **호출되는 에이전트**: Direction Auditor 1회.
- **필요한 LLM 호출 수**: 1회.
- **streaming 사용 시 동작**: 완성 후 호출. 스트리밍 중에는 audit-only.
- **UI 설정**: "감정선 강도 (부드러움/극적)", "반전 허용 임계값".
- **구현 난이도**: 중.
- **MVP / 추후 버전 / 실험 기능**: 추후 버전.
- **기존 플러그인과의 차별점**: **감정의 흐름을 시각화/수치화하고 연출적으로 개선**.

---

### 6. 장면 블로킹 & 호흡 조율관 (Scene Blocking & Pacing Composer)

- **목적**: 등장인물의 물리적 위치, 동작 순서, 장면의 호흡(긴장/완화)이 모순되지 않는지 확인.
- **사용자 체감 효과**: "방금 앉았는데 서 있다", "전투가 너무 길어졌다/짧아졌다" 방지.
- **동작 흐름**:
  1. 응답에서 등장인물, 위치, 동작, 시간 흐름 추출.
  2. 물리적 모순(위치, 자세)과 호흡 모순(너무 급격한 전환) 감지.
  3. bounded patch 제안.
- **호출되는 에이전트**: Direction Auditor 1회.
- **필요한 LLM 호출 수**: 1회.
- **streaming 사용 시 동작**: 완성 후 호출.
- **UI 설정**: "장면 블로킹 검사 on/off", "호흡 조절 강도".
- **구현 난이도**: 중~상.
- **MVP / 추후 버전 / 실험 기능**: 추후 버전.
- **기존 플러그인과의 차별점**: **문장 수준이 아니라 장면의 공간적/시간적 일관성을 검사**.

---

### 7. 보호 영역 봉인 수호자 (Protected Envelope Guardian)

- **목적**: 이미지 태그, 시스템 태그, 코드블록, 특수 마크업, 숨겨진 프롬프트를 절대 수정하지 않도록 보호.
- **사용자 체감 효과**: 이미지 생성 태그가 깨지거나, 모듈 태그가 날아가는 사고 방지.
- **동작 흐름**:
  1. 출력을 보호 영역(prefix/suffix/inline)과 가변 본문으로 분리.
  2. 모든 patch는 가변 본문에만 적용.
  3. 복원 후 보호 영역이 변형되면 전체 결과 폐기.
- **호출되는 에이전트**: rule-based + lightweight LLM optional.
- **필요한 LLM 호출 수**: 0회 (rule-based).
- **streaming 사용 시 동작**: 모든 모드에서 항상 동작.
- **UI 설정**: "보호 마커 목록" (기본값 제공, 사용자 추가 가능).
- **구현 난이도**: 중.
- **MVP / 추후 버전 / 실험 기능**: MVP.
- **기존 플러그인과의 차별점**: **후처리 플러그인이 종종 깨뜨리는 이미지/시스템 태그를 완전히 분리 보호**.

---

### 8. 에이전트 충돌 중재관 (Agent Conflict Arbiter)

- **목적**: 여러 AI의 patch 제안이 충돌할 때 우선순위와 병합 규칙을 적용.
- **사용자 체감 효과**: "AI들이 서로 다른 말을 해서 더 이상해졌다" 방지.
- **동작 흐름**:
  1. 모든 에이전트의 제안을 수집.
  2. 우선순위(캐릭터 > 세계관 > 플롯 > 문체 > 연출)와 confidence로 필터.
  3. 동일 구간 충돌 시 우선순위 적용, 해결 불가능 시 원문 유지.
- **호출되는 에이전트**: Meta-arbiter (rule-based, LLM optional).
- **필요한 LLM 호출 수**: 0~1회.
- **streaming 사용 시 동작**: 완성 후 호출.
- **UI 설정**: "우선 모드 (설정/캐릭터/감정선/문체)".
- **구현 난이도**: 중.
- **MVP / 추후 버전 / 실험 기능**: 추후 버전 (MVP에서는 2개 에이전트만으로 충돌 최소화).
- **기존 플러그인과의 차별점**: **다중 에이전트의 의견을 통합하는 공식 규칙을 가짐**.

---

### 9. 상황별 에이전트 라우터 (Context-Aware Agent Router)

- **목적**: 응답의 특성에 따라 필요한 에이전트만 호출해 비용과 지연을 줄임.
- **사용자 체감 효과**: 빠른 응답에서도 품질 개선을 느낌. 비용 부담 감소.
- **동작 흐름**:
  1. 응답의 길이, 대화 비율, 장면 전환 여부, 이미지 태그 존재 여부를 분석.
  2. 라우팅 규칙에 따라 에이전트 선택.
  3. 선택된 에이전트만 병렬 호출.
- **호출되는 에이전트**: Router (rule-based) + 선택된 전문 에이전트들.
- **필요한 LLM 호출 수**: 0~5회 (상황에 따라).
- **streaming 사용 시 동작**: 완성 후 라우팅.
- **UI 설정**: "자동 에이전트 선택 on/off", "최대 동시 호출 수".
- **구현 난이도**: 중.
- **MVP / 추후 버전 / 실험 기능**: 추후 버전.
- **기존 플러그인과의 차별점**: **모든 검사를 항상 돌리는 것이 아니라 상황에 맞게 최소한의 검사만 수행**.

---

### 10. 스트리밍 인식 출력 감사 (Streaming-Aware Output Auditor)

- **목적**: 스트리밍 중에는 부분 출력을 audit-only로 점검하고, 완성 후에만 patch 적용.
- **사용자 체감 효과**: 스트리밍 사용 시에도 품질 감독이 동작하는 느낌. 단, 출력이 끊기지 않음.
- **동작 흐름**:
  1. 스트리밍 중: 마지막 chunk가 도착할 때마다 lightweight audit (Character/Style only).
  2. 완성 시점: 전체 응답에 대해 전체 패널 실행.
  3. patch는 완성 후에만 적용.
- **호출되는 에이전트**: 스트리밍 중 Character/Style, 완성 후 전체.
- **필요한 LLM 호출 수**: 0~2회 (스트리밍 중) + 1~5회 (완성 후).
- **streaming 사용 시 동작**: 핵심 기능.
- **UI 설정**: "스트리밍 모드 (audit-only / 완성 후 patch / 비활성)".
- **구현 난이도**: 상.
- **MVP / 추후 버전 / 실험 기능**: 추후 버전.
- **기존 플러그인과의 차별점**: **스트리밍 환경에서도 안전하게 동작하는 감독 파이프라인**.

---

### 11. 이미지 태그 일관성 감사 (Image Tag Consistency Auditor)

- **목적**: 이미지 생성 태그가 캐릭터/세계관/장면과 모순되지 않는지 점검.
- **사용자 체감 효과**: "그림은 다른 옷을 입고 있다" 같은 시각-텍스트 불일치 방지.
- **동작 흐름**:
  1. 이미지 태그를 추출.
  2. 태그의 등장인물, 의상, 배경, 분위기가 본문과 일치하는지 LLM으로 검사.
  3. 불일치 시 태그 수정 제안 (보호 영역 내에서만).
- **호출되는 에이전트**: Character/World/Style Auditor 중 1~2개.
- **필요한 LLM 호출 수**: 1회.
- **streaming 사용 시 동작**: 완성 후 호출.
- **UI 설정**: "이미지 태그 검사 on/off".
- **구현 난이도**: 중.
- **MVP / 추후 버전 / 실험 기능**: 실험 기능.
- **기존 플러그인과의 차별점**: **텍스트뿐 아니라 이미지 생성 태그의 일관성까지 감독**.

---

### 12. OOC / 메타 누설 감지기 (OOC & Meta Leak Detector)

- **목적**: 캐릭터가 플레이어, AI, 시스템에 대해 언급하거나, 현실 세계 개념을 RP 세계로 끌어오는 것을 감지.
- **사용자 체감 효과**: "캐릭터가 갑자기 '플레이어'라고 부름", "현대 용어가 튀어나옴" 방지.
- **동작 흐름**:
  1. 응답에서 OOC 징후(메타 언급, 현실 세계 개념, 시스템 언급) 추출.
  2. 캐릭터/세계관 에이전트와 병합 판단.
  3. bounded patch 제안 또는 reject.
- **호출되는 에이전트**: Character + World Auditor.
- **필요한 LLM 호출 수**: 1회 (병합 호출).
- **streaming 사용 시 동작**: 완성 후 호출.
- **UI 설정**: "OOC 감지 강도", "메타 언급 차단".
- **구현 난이도**: 하~중.
- **MVP / 추후 버전 / 실험 기능**: MVP.
- **기존 플러그인과의 차별점**: **단순히 "OOC다"가 아니라, 캐릭터/세계관 기준에서 왜 OOC인지 설명하고 patch 제안**.
