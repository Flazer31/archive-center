# Archive Center 2.5 MDASH Output Harness Plan

Status: planned target after 2.2 stabilization

Direction update:

`2.5-standalone-output-quality-layer-plan.md` is now the active design anchor
for the 2.5 output work. The first implementation should be a standalone RisuAI
output quality layer. Archive Center connection is optional and deferred. This
file remains as the historical MDASH/Table Read harness reference and should be
read through that newer standalone-first boundary.

This document fixes the 2.5 direction for the MDASH-inspired output quality
lane. It combines pre-draft guidance and post-draft review into one bounded
harness, and keeps the implementation separable from the main `Archive
Center.js` runtime.

## Goal

2.5 should add an optional high-attention output lane:

```text
raw user input + Archive Center memory packet
-> pre-draft guidance
-> main LLM draft
-> output MDASH / Table Read review
-> patch planner
-> code-side patch apply
-> verifier
-> final output or original fallback
```

The feature is not a replacement for normal memory, recall, storage, or
rollback. It must remain opt-in and fail-open.

## Non-Negotiable Rules

- Do not rewrite the user's raw input as truth.
- Do not make Table Read or MDASH a memory writer.
- Do not let reviewer agents directly replace the final response.
- Do not run hidden expensive calls in the normal path.
- Do not drop image tags, status blocks, Chatindex lines, regex-facing markers,
  module tags, or card-specific output structures.
- If verifier fails, return the original main LLM draft.
- The accepted final output is saved through the normal Archive Center turn
  save path.

## Independent Satellite Plugin Strategy

The MDASH output harness should not be forced into one giant Archive Center
plugin or one giant MDASH plugin.

2.5 should use independent optional satellite plugins. Each satellite can run
by itself in RisuAI, and when Archive Center is reachable it detects the
connection and enriches its work with Archive Center memory/state data.

Recommended package names:

```text
Archive Center.js                   # core bridge, memory, dashboard, capture
Archive Center Guidance.js           # optional pre-draft guidance satellite
Archive Center Output Check.js       # optional output inspection satellite
Archive Center Table Read.js         # optional Table Read satellite
Archive Center Patch Guard.js        # optional patch/apply/verifier satellite
```

This prevents `Archive Center.js` from becoming a giant all-in-one runtime and
also prevents MDASH from becoming one heavy mandatory bundle.

No satellite should require another satellite to function. A later convenience
bundle may exist, but the product architecture is independent modules first.

Each optional satellite should support two modes:

```text
connected_archive_center
standalone_risuai
```

### connected_archive_center

This is the enhanced Archive Center-connected mode.

A satellite may call the Archive Center backend for:

- selected memory packet
- subjective entity memory
- character states
- world rules
- relationship state
- scene participants
- output trace storage
- verifier helper routes

The satellite must treat Archive Center as a data provider and verification
partner, not as a place to outsource uncontrolled writes. It must not write
canonical memory by itself.

### standalone_risuai

This mode lets each satellite JS file be used without the Archive Center
backend.

It can use:

- current RisuAI messages
- character description/persona/lore if available through RisuAI APIs
- plugin-local settings
- provider presets
- pre-draft guidance
- output check
- patch planning
- verifier guards that can run in JS

It cannot provide the full Archive Center value:

- no MariaDB canonical memory
- no Chroma/vector recall
- no subjective entity memory from Archive Center
- no rollback/delete consistency
- no Archive Center timeline/explorer trace

The UI must show this distinction clearly:

```text
MDASH mode: standalone
Archive memory: unavailable
Subjective character memory: unavailable
```

Standalone mode is useful, but it is not equivalent to full Archive Center.

## Module Shape

Each satellite JS file should avoid depending on hidden globals from
`Archive Center.js` or from other satellites. Shared behavior should be
expressed through stable contracts and small duplicated adapters where needed.

Suggested shared conceptual layout per satellite:

```text
SatelliteCore
  - mode planner
  - role registry for that satellite only
  - prompt builders
  - structured output parsers
  - issue merge when relevant
  - patch planner when relevant
  - verifier when relevant

RisuAIAdapter
  - beforeRequest hook
  - afterRequest/output hook
  - current chat extraction
  - auxiliary request bypass
  - protected segment detection

ArchiveCenterAdapter
  - backend health/readiness
  - prepare-turn memory packet read
  - entity/subjective memory read
  - output trace write
  - optional verifier helper calls

ProviderRegistry
  - OpenAI-compatible
  - Anthropic/Claude
  - Gemini/Vertex
  - Ollama/Ollama Cloud
  - custom endpoint

RunLedger
  - requested mode
  - actual mode
  - roles run by this satellite
  - provider calls
  - downgrade reason
  - verifier result
```

The Archive Center adapter contract should be HTTP/API based where possible so
that satellites can discover the core backend without requiring direct import
or shared JS runtime state.

## Pre-Draft Guidance

Pre-draft guidance is the front half of the feature. It runs before the main
LLM draft and gives the main model acting/directing guidance without replacing
the user's input.

Inputs:

- raw user input
- recent chat
- selected Archive Center memory packet when connected
- scene participants
- relevant character states
- relevant world rules
- user-visible constraints

Outputs:

- scene director brief
- character director brief
- plot brief
- world brief
- continuity brief
- secret/POV brief
- memory survival brief

The brief is inserted as support context only.

It must not become:

- rewritten user input
- canonical truth
- DB write candidate
- hidden forced plot result

## Main Draft

The main LLM produces the actual assistant draft.

In RisuAI technical terms, this may be hard to intercept perfectly before
display when streaming is enabled. If true pre-display interception is not
available, the MDASH output lane must degrade to audit/trace or explicit
non-streaming opt-in mode. The UI must not pretend a streamed raw draft was
cleanly improved before display.

## Output MDASH Review

The post-draft half reviews the assistant draft.

Specialized reviewer roles:

- Character Director: overall ensemble balance, role distribution, dialogue
  order, relationship flow.
- Individual Character Reader: one character's voice, knowledge boundary,
  subjective memory, emotion, private recollection, and secret boundary.
- Plot Reader: scene goal, tension, pacing, next-turn openness.
- World Reader: world rules, physical/social law, setting constraints,
  location and time.
- Continuity Reader: recent memory, long memory, relationship shifts, repeated
  incident drift.
- Secret Guard: hidden/private knowledge leakage, meta leakage, reincarnation
  or loop-related leak risk.
- Structure Guard: image tags, status windows, Chatindex, regex markers,
  module/helper tags, non-prose blocks.

Reviewer agents return findings, not replacement prose.

Example finding shape:

```json
{
  "role": "world_reader",
  "verdict": "needs_patch",
  "confidence": 0.78,
  "evidence_refs": ["paragraph_4"],
  "authority_lane_used": ["world_rules", "recent_chat"],
  "issues": [
    {
      "type": "world_rule_conflict",
      "detail": "The draft treats a scarce resource as freely available."
    }
  ],
  "recommended_action": "patch_mutable_segment",
  "risk_flags": []
}
```

## Full Table Read

Full Table Read is the high-cost character review mode.

It should split the character lane into:

```text
Character Director
-> Individual Character Readers
-> Character Director Round 2
```

For `C` active characters, the character-only part is:

```text
initial director agenda: 1 call
first character opinions: C calls
director summary: 1 call
cross-character responses: C calls
final character direction: 1 call
```

That is approximately:

```text
2C + 3 calls
```

The broader historical design estimated `2C + 5` for the Table Read portion
when issue sorting and scene director conclusion are included.

Full 2.5 output MDASH with output check, plot/world/continuity/secret readers,
patch planner, and verifier can reach:

```text
2C + 7 to 9 additional LLM calls
```

This must be important-scene / manual / high-quality opt-in only.

## Patch Planner

The patch planner merges all reviewer findings.

It must produce minimal patch objects, not a full replacement response.

Patch shape:

```json
{
  "segment_id": "mutable_3",
  "operation": "replace",
  "replacement": "new text for this mutable segment only",
  "reason": "remove private-memory leakage",
  "source_findings": ["secret_guard_1", "character_reader_lia_2"]
}
```

Archive Center code applies patches only to mutable segments.

## Verifier

The verifier has veto power, not creative rewrite power.

Verifier checks:

- protected segments unchanged
- no image/status/regex/module marker removed
- paragraph retention is acceptable
- length ratio is not suspicious
- no hidden/private memory leaked as objective fact
- no unsourced new fact added
- no user-requested content removed
- no output truncation
- same language/format preserved unless explicitly requested otherwise

If verifier fails:

```text
return original draft
record failure reason
do not save reviewer text as memory
```

## Execution Modes

| Mode | Extra LLM calls | Behavior |
|---|---:|---|
| `off` | 0 | Normal Archive Center behavior |
| `pre_guidance` | 1 | Pre-draft guidance only |
| `check_only` | 1 | Output check only; no patch |
| `patch_light` | 1-2 | Output check plus patch planner when needed |
| `single_panel` | 2-3 | One simulated multi-role panel plus patch planner |
| `full_table_read` | `2C + 7~9` | Full character Table Read plus specialist readers |

The user selects the maximum mode. Archive Center may downgrade when cost,
latency, provider availability, streaming mode, or route readiness is not green.

## 2.5 Implementation Order

1. `2.5-0` Satellite contract shell
   - Define the satellite plugin contract.
   - Implement shared-but-copyable RisuAI hook, settings, Archive Center probe,
     mode display, and fail-open pass-through pattern.
   - Do not create one monolithic MDASH runtime.

2. `2.5-1` Adapter contracts
   - Define connected Archive Center mode and standalone mode.
   - Add backend capability probe.
   - Add auxiliary request bypass for memory, translate, image, module,
     submodel, OtherAx, Lightboard/lb-process-like helper calls.

3. `2.5-2` Pre-draft guidance
   - Add `Archive Center Guidance.js`.
   - Add support-only scene/character/plot/world/secret guidance.
   - Ensure raw user input is never replaced.

4. `2.5-3` Output check and structure guard
   - Add `Archive Center Output Check.js`.
   - Segment draft into protected, mutable, and inspect-only parts.
   - Add output issue scan.
   - No output modification yet.

5. `2.5-4` Patch planner and JS verifier
   - Add `Archive Center Patch Guard.js`.
   - Add minimal patch contract.
   - Apply patches only to mutable segments.
   - Fallback to original on verifier failure.

6. `2.5-5` Single-panel Table Read
   - Add `Archive Center Table Read.js`.
   - One model call simulates Character Director, Plot Reader, World Reader,
     Continuity Reader, Secret Guard, and Structure Guard.
   - Mark as simulated panel, not independent multi-agent verification.

7. `2.5-6` Full Table Read
   - Add Character Director plus Individual Character Readers.
   - Add cross-character response round.
   - Keep participant count and call budget explicit.

8. `2.5-7` Independent verifier and trace UI
   - Optional separate verifier model.
   - Show requested mode, actual mode, calls used, downgrade reason, patches
     accepted/rejected, and fallback reason.

9. `2.5-8` Adoption gate
   - Replay image-tag preservation.
   - Replay streaming/non-streaming.
   - Replay hidden-info leak.
   - Replay character voice drift.
   - Replay world-rule contradiction.
   - Replay long-output truncation.

## Packaging Policy

The 2.5 package should include the MDASH satellites as optional files:

```text
Archive Center.js
Archive Center Guidance.js
Archive Center Output Check.js
Archive Center Table Read.js
Archive Center Patch Guard.js
```

Default install should enable only the core plugin.

Each satellite should show:

```text
Disabled by default
Extra LLM calls may occur when enabled
Standalone mode has reduced memory awareness
Connected mode requires Archive Center backend readiness
```

## Relationship To Example Plugins

Reference ideas from `risu_agents.js` and
`Serial_Gradation_Agents_for_RP.js` are useful:

- pre-agent guidance
- role separation
- provider presets
- run inspector
- auxiliary request bypass
- fail-open behavior

Archive Center must not copy:

- full afterRequest replacement as product behavior
- unrestricted post-agent polish
- plugin-local memory as canonical truth
- serial draft rewriting as the main Table Read mechanism

Archive Center's differentiator is connected memory authority:

- MariaDB canonical data
- vector recall
- subjective entity memory
- rollback/delete/session consistency
- output verifier and normal save path

## Done Criteria

- Core chat still works when MDASH is off.
- MDASH failure never blocks the main output.
- Raw user input remains unchanged and inspectable.
- Protected output blocks survive every accepted patch.
- Standalone mode is usable but clearly labeled as reduced capability.
- Connected mode can use Archive Center memory without granting reviewers DB
  write authority.
- Full Table Read is opt-in and budget-limited.
- Each satellite can be enabled, disabled, and tested independently.
- Satellite-to-satellite dependency is optional, not required.
- The main `Archive Center.js` file does not absorb the whole MDASH runtime.
