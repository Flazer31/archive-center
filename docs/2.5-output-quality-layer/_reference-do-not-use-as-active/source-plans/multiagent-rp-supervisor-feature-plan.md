# 멀티에이전트 RP 출력 감독 시스템 — 기능 기획안 (2.5 Output Quality Layer)

> 범위 한정: 이 문서는 **MDASH식 멀티에이전트 출력 개선**만 다룬다. Archive Center / DB / 장기기억 시스템에 의존하지 않으며, 사용자의 입력문을 수정하지 않고, "전체 무조건 재작성"을 회피하는 bounded patch 중심 설계다. 안전성 경고가 아닌 **기능 발전 방향** 중심으로 정리한다.

---

## 0. 한 줄 정의

RisuAI의 메인 모델이 만든 RP/소설형 응답 초안을, 역할이 분리된 감독 에이전트들이 **점검 → 충돌 조정 → 문제 구간만 국소 수정**하는 출력 감독 레이어.

핵심 명제: "교정기"가 아니라 **편집 감독실(writers' room)** 이다.

---

## 1. 기존 후처리/agents 플러그인 대비 차별점

기존 플러그인은 대부분 (a) 단순 텍스트 후처리(정규식/문법 교정), 또는 (b) "응답을 통째로 다시 써줘" 식 단일 리라이트 에이전트다. 이 플러그인의 차별점은 다음과 같다.

| 축 | 기존 후처리/리라이트 플러그인 | 본 플러그인(감독 시스템) |
|---|---|---|
| 수정 단위 | 전체 응답 재생성 | **문제 구간만 bounded patch** |
| 판단 주체 | 단일 모델 1회 | **역할 분리 감독진 + 최종 Director** |
| 점검 차원 | 문법/길이 | 캐릭터·세계관·플롯·문체·연출 **다축 동시 점검** |
| 실패 처리 | 망가진 출력 그대로 노출 | **확신 낮으면 원문 유지(fail-safe revert)** |
| 보호 영역 | 태그/마크업 깨짐 빈번 | 이미지/시스템/숨김프롬프트/코드블록 **마스킹 보존** |
| 입력 취급 | 가끔 유저 입력까지 손댐 | **유저 입력 절대 불변** |
| 투명성 | 결과만 교체 | **무엇을 왜 고쳤는지 감독 로그/diff 제공** |

핵심 차별점 3가지로 압축:
1. **국소 수정(patch) 우선** — "고칠 만큼만 고친다". 응답의 톤과 분량을 살린다.
2. **다역할 감독 + 조정** — 단일 관점이 아니라 캐릭터/세계관/플롯/문체/연출의 충돌을 명시적으로 조정한다.
3. **보수적 fail-safe** — 개선 확신이 임계치 미만이면 차라리 손대지 않는다. "개악"을 구조적으로 막는 것이 신뢰의 핵심.

## 2. MDASH 강점을 살리기 위한 핵심 기능

MDASH식(여러 특화 에이전트가 동일 산출물을 다른 관점에서 점검 → 통합)의 진짜 강점은 **"관점 분리로 인한 발견율 향상"** 과 **"통합자에 의한 선택적 채택"** 이다. 이를 살리려면:

### 2.1 구간 단위 어노테이션 프로토콜 (Span Annotation Protocol)
- **목적:** 에이전트가 "전체 다시 써" 대신 `{구간 위치, 문제 유형, 심각도, 제안 패치}` 구조로만 응답하도록 강제.
- **사용자 체감 효과:** 응답이 통째로 바뀌지 않고 어색한 부분만 자연스럽게 다듬어진다.
- **동작 흐름:** 초안 → 각 에이전트가 JSON 어노테이션 리스트 반환 → Director가 병합/적용.
- **호출되는 에이전트:** 전 역할 공통(출력 포맷 규약).
- **필요한 LLM 호출 수:** 에이전트 수 + Director 1회.
- **streaming 사용 시 동작:** 스트림 완료 후 일괄 어노테이트(중간 점검 금지).
- **UI 설정:** 어노테이션 표시 on/off, 심각도 임계치 슬라이더.
- **구현 난이도:** 중. (출력 스키마 강제 + 위치 매칭이 관건)
- **MVP / 추후 / 실험:** **MVP**.
- **기존 플러그인과의 차별점:** 통짜 재작성이 아닌 구조화된 패치 단위 — 이 플러그인의 뼈대.

### 2.2 보호 마스킹 파이프라인 (Protected Span Masking)
- **목적:** 이미지 태그, 시스템 태그, 숨김 프롬프트, 코드블록, 특수 마크업을 점검 전에 플레이스홀더로 치환해 에이전트가 손대지 못하게 하고, 패치 적용 후 복원.
- **사용자 체감 효과:** `<img>`, `{{...}}`, ```code``` 등이 절대 깨지지 않는다.
- **동작 흐름:** 마스킹 → 에이전트 점검(마스크 구간은 read-only) → 패치 적용 → 언마스킹/무결성 검증.
- **호출되는 에이전트:** 없음(전처리/후처리 로직).
- **필요한 LLM 호출 수:** 0.
- **streaming 사용 시 동작:** 완료 후 마스킹.
- **UI 설정:** 보호 패턴 목록 편집(고급).
- **구현 난이도:** 중. (마스킹 매핑 + 복원 검증 필수)
- **MVP / 추후 / 실험:** **MVP** (안전성의 전제조건).
- **기존 플러그인과의 차별점:** 후처리 플러그인의 고질병(태그 깨짐)을 구조적으로 차단.

### 2.3 패치 적용기 + 무결성 게이트 (Patch Applier & Integrity Gate)
- **목적:** 어노테이션을 실제 텍스트에 안전하게 반영하고, 적용 결과가 (a) 보호구간 보존, (b) 분량 급변 없음, (c) 마크업 균형을 통과해야만 확정.
- **사용자 체감 효과:** 수정 후에도 문서 구조가 안정적.
- **동작 흐름:** 패치 후보 → 게이트 검증 → 통과 시 확정 / 실패 시 해당 패치 폐기(부분 revert).
- **호출되는 에이전트:** 없음.
- **필요한 LLM 호출 수:** 0.
- **streaming 사용 시 동작:** 무관(후단).
- **UI 설정:** 분량 변동 허용 폭(%) 설정.
- **구현 난이도:** 중상.
- **MVP / 추후 / 실험:** **MVP**.
- **기존 플러그인과의 차별점:** "개악 방지"를 코드로 보장.

### 2.4 감독 트레이스 / Diff 뷰 (Audit Trace)
- **목적:** 어떤 에이전트가 무엇을 왜 고쳤는지 before/after diff와 사유를 남김.
- **사용자 체감 효과:** 신뢰 + 디버깅. 마음에 안 들면 해당 패치만 되돌릴 수 있음.
- **동작 흐름:** 패치마다 메타(에이전트, 사유, 심각도) 기록 → 접이식 뷰.
- **호출되는 에이전트:** 없음.
- **필요한 LLM 호출 수:** 0.
- **streaming 사용 시 동작:** 무관.
- **UI 설정:** 트레이스 패널 표시 토글.
- **구현 난이도:** 중.
- **MVP / 추후 / 실험:** MVP(최소형) → 추후 인터랙티브 revert.
- **기존 플러그인과의 차별점:** 블랙박스가 아닌 설명가능한 편집.


## 3~7. 역할별 감독 에이전트 점검 항목

> 공통 원칙: 모든 역할은 **자기 관할(domain)만** 어노테이션한다. 관할 밖은 보지 않는다. 이것이 충돌 조정을 가능하게 한다. 각 역할은 응답 초안 + 컨텍스트(캐릭터 카드, 직전 N턴, 시스템 노트)를 입력으로 받되, **DB나 장기기억이 아니라 현재 대화 컨텍스트 윈도우만** 근거로 한다.

### 3. 캐릭터 담당 AI (Character Steward)
점검 항목:
- **말투/1인칭/호칭 일관성:** 캐릭터 카드에 정의된 어조, 반말/존댓말, 상대 호칭이 응답에서 유지되는가.
- **성격·가치관 위반(OOC) 탐지:** 겁많은 캐릭터가 갑자기 무모, 냉정한 캐릭터가 갑자기 감상적 등 급격한 인격 붕괴.
- **지식 경계 위반:** 캐릭터가 알 수 없는 정보를 아는 것처럼 말함(메타 누설, 4번째 벽).
- **관계 톤:** 상대 캐릭터에 대한 태도가 직전 맥락 대비 부자연스럽게 점프하지 않는가.
- **신체/외형 모순:** 카드에 명시된 외형/능력과 즉시 충돌하는 묘사.
- **자기 발화 vs 행동 일치:** 말과 행동(액션 비트)이 캐릭터답게 맞물리는가.
패치 방향: 톤/호칭은 단어 단위 치환, OOC는 해당 대사/행동 한 문장 국소 교정. 광범위 성격 재해석 금지.

### 4. 세계관 담당 AI (Lore/World Steward)
점검 항목:
- **설정 사실 위반:** 카드/로어북/시스템 노트에 명시된 사실(지명, 종족 규칙, 마법/기술 제약)과의 직접 충돌.
- **시대/톤 위배:** 중세 판타지에 현대 슬랭/기기, 하드SF에 마법 등 장르 톤 누수.
- **명칭 표기 일관성:** 고유명사 철자/표기 흔들림(같은 인물·장소를 다른 이름으로).
- **물리/공간 모순:** 방금 "밖은 한밤중"인데 "햇살" 같은 환경 모순.
- **제약 위반:** 세계 규칙상 불가능한 사건(죽은 자가 말함 등)이 설명 없이 발생.
패치 방향: 사실 충돌 어구만 치환 또는 한 줄 추가 보정. 세계관 재설계 금지. **불확실하면 "변경 제안"이 아니라 "플래그만"** 띄우고 Director 판단에 위임.

### 5. 플롯/서사 담당 AI (Plot/Continuity Steward)
점검 항목:
- **직전 사건 연속성:** 바로 앞 턴에서 일어난 일(부상, 이동, 약속)을 무시하지 않는가.
- **인과 단절:** 동기 없이 행동이 점프, 갑툭튀 사건.
- **장면 목표 표류:** 현재 씬이 향하던 방향에서 이유 없이 이탈.
- **유저 행동 무시(agency 침해):** 유저가 한 선택/행동을 응답이 무시하거나 강제로 덮어쓰는가(가장 중요한 RP 불만 요인).
- **반복/제자리걸음:** 직전 턴과 의미상 동일한 진행(루프).
- **미해결 떡밥 방치 여부(약한 신호):** 직전 턴이 던진 질문/긴장이 통째로 증발했는가.
패치 방향: 연결 문장 1~2개 삽입 또는 모순 행동 국소 수정. 새 플롯 발명 금지. 유저 agency 침해는 **최우선 심각도**.

### 6. 문체/문장 담당 AI (Prose/Style Steward)
점검 항목:
- **반복 표현/상투구:** 같은 단어·문형·클리셰("심장이 쿵") 과다.
- **문장 리듬:** 단조로운 길이, 어색한 호흡, 비문/꼬인 구문.
- **시점/시제 일관성:** 1인칭/3인칭, 과거/현재 시제 혼선.
- **번역체/AI체 냄새:** 부자연스러운 직역투, 과도한 부사, "~할 수 있었다" 남발.
- **묘사/대사 비율:** 대사만 나열 또는 묘사 과잉으로 호흡 붕괴.
- **포맷 일관성:** 행동은 *별표*, 대사는 따옴표 등 사용자의 기존 포맷 관습 유지.
패치 방향: 가장 안전한 국소 치환 역할. 단, **의미 변경 금지**(표현만 다듬기). 분량 동결.

### 7. 연출/감정선 담당 AI (Direction/Emotion Steward)
점검 항목:
- **감정 연속성:** 직전 감정 상태에서 근거 없는 급변(분노→갑자기 평온).
- **긴장 곡선/페이싱:** 클라이맥스를 너무 빨리 소진하거나 김 빠지는 마무리.
- **감각 연출 결핍/과잉:** 핵심 순간에 오감 디테일 부재, 혹은 사소한 장면에 과한 연출.
- **장면 전환 매끄러움:** 컷 전환이 급작스럽거나 공간/시간 점프 표시 부재.
- **여운/훅:** 턴 마무리가 다음 행동을 유도하는 여지를 남기는가(특히 RP 몰입).
- **감정-행동 정합:** 감정 상태와 행동/대사 톤이 어긋나지 않는가.
패치 방향: 연출은 "추가/강화"가 많아 분량 증가 위험 → **삽입은 짧은 비트로 제한**, 의미상 새 사건 추가 금지.

## 8. 충돌 시 최종 결정자(Director) 동작

Director는 추가 의견을 내는 6번째 비평가가 아니라 **편집장(중재자 + 적용 결정자)** 이다.

### 동작 원칙
1. **관할 우선권(domain priority):** 같은 구간을 두 역할이 다르게 고치려 하면, 그 구간의 **의미 소유 역할이 우선**한다.
   - 의미/사실 충돌(캐릭터·세계관·플롯)은 항상 문체/연출보다 우선.
   - 예: 문체 담당이 "더 시적으로" 바꾸려는 문장이 캐릭터 담당이 "OOC"로 표시한 문장이면 → 캐릭터 수정이 우선, 문체는 그 결과 위에서만 다듬음.
2. **심각도 게이팅:** 각 어노테이션은 심각도(예: blocker / major / minor)를 갖는다. blocker·major만 기본 적용, minor는 모드에 따라 선택 적용.
3. **충돌 해소 순서:** 사실축(캐릭터→세계관→플롯) 패치 먼저 확정 → 그 위에 표현축(연출→문체) 패치를 재정렬(겹치면 표현축 패치를 사실축 결과에 맞게 재매핑하거나 폐기).
4. **상호 모순 패치 폐기:** 두 사실축 역할이 서로 양립 불가한 변경을 요구하면, Director는 **더 보수적인(원문 보존적인) 쪽** 또는 **둘 다 폐기 후 플래그만 표시**.
5. **분량/톤 예산:** 누적 패치가 분량 변동 한도를 넘으면 minor부터 잘라낸다.
6. **확신 임계치 미달 → 전체 revert:** 적용 후에도 게이트 통과가 불안하거나 패치 신뢰도 합이 낮으면 **원문 응답을 그대로 둔다**.

### 동작 모드
- **단일 통합 호출(권장 MVP):** 모든 어노테이션을 모아 Director에게 1회 호출 → 우선순위 규칙대로 최종 패치 세트 산출.
- **규칙 기반 무LLM 병합(저비용 옵션):** 관할/심각도/겹침만으로 결정론적 병합(LLM 0회). 빠르지만 미세 조정 약함.

### 충돌 처리 기능 카드 — Director Arbiter
- **목적:** 다역할 어노테이션을 단일 일관 패치 세트로 통합.
- **사용자 체감 효과:** 모순된 수정 없이 깔끔한 한 버전.
- **동작 흐름:** 어노테이션 수집 → 관할/심각도 정렬 → 겹침 해소 → 게이트 → 확정 or revert.
- **호출되는 에이전트:** Director.
- **필요한 LLM 호출 수:** 1 (또는 0, 규칙 병합 모드).
- **streaming 사용 시 동작:** 스트림 종료 후 1회.
- **UI 설정:** 충돌 해소 정책(보수/적극), 심각도 임계치.
- **구현 난이도:** 중상.
- **MVP / 추후 / 실험:** **MVP**.
- **기존 플러그인과의 차별점:** 멀티에이전트 의견을 "투표"가 아니라 **관할 기반 중재**로 통합.

## 9. 비용/속도 최적화: 선택적 에이전트 호출

전부 호출은 느리고 비싸다. **트리아지(triage)** 로 필요한 역할만 깨운다.

### 9.1 경량 트리아지 게이트 (Triage Gate)
- **목적:** 초안을 값싼 휴리스틱/소형 모델로 스캔해 "어떤 도메인에 문제 징후가 있는지"만 판단 → 해당 역할만 호출.
- **사용자 체감 효과:** 평소엔 빠르고 싸고, 문제 있을 때만 깊게 점검.
- **동작 흐름:** 초안 → 트리아지(신호 탐지) → 후보 역할 집합 결정 → 그 역할만 어노테이트.
- **호출되는 에이전트:** 트리아지(소형) + 선택된 역할들 + Director.
- **필요한 LLM 호출 수:** 1(트리아지) + N(선택) + 1(Director). 무패치면 조기 종료.
- **streaming 사용 시 동작:** 완료 후.
- **UI 설정:** "자동 트리아지" on/off, 항상 호출할 역할 고정.
- **구현 난이도:** 중.
- **MVP / 추후 / 실험:** MVP(휴리스틱) → 추후 소형 모델 트리아지.

### 트리아지 신호 예시(무LLM 휴리스틱)
- 반복 n-그램/상투구 빈도 ↑ → 문체 호출.
- 고유명사 표기 흔들림/카드 키워드 충돌 → 세계관 호출.
- 호칭/시제/시점 급변 → 캐릭터 호출.
- 직전 턴 키 사건 키워드 부재 → 플롯 호출.
- 감정어 급반전 → 연출 호출.
- 신호 없음 → **모든 역할 스킵, 원문 그대로**(가장 흔한 빠른 경로).

### 9.2 조기 종료 / 예산 캡 (Early Exit & Budget Cap)
- **목적:** 패치 후보가 0이거나 비용 상한 도달 시 즉시 중단.
- **필요한 LLM 호출 수:** 가변(상한 설정).
- **MVP / 추후 / 실험:** MVP.

### 호출 절약 규칙 요약
- 짧은 응답(예: 단답/한 줄)은 문체만 또는 전체 스킵.
- 동일 씬 연속 턴이면 세계관 점검 빈도 다운샘플.
- "확신 높은 깔끔한 초안"은 트리아지에서 통과시켜 0회 LLM.

## 10. 사용자 UI 모드

사용자는 "얼마나 깊게 / 언제 / 무엇을" 감독할지 선택할 수 있어야 한다.

### 감독 강도 모드
- **Off:** 감독 비활성(순정 출력).
- **Audit-only(관전 모드):** 응답은 **건드리지 않고** 어노테이션/지적만 패널에 표시. streaming 친화적, 비파괴. 학습/디버깅용.
- **Light(가벼운 다듬기):** 문체 + 명백한 오류(blocker)만 국소 패치. 1~2 에이전트.
- **Standard(권장 기본):** 트리아지 후 필요한 역할만 + Director. 균형.
- **Deep(정밀 감독):** 전 역할 호출 + Director. 비싸고 느림. 중요 장면용.

### 트리거 모드
- **자동(every turn):** 매 응답마다 감독.
- **온디맨드(버튼):** 사용자가 "이 응답 다듬기" 버튼 누를 때만.
- **임계 자동:** 트리아지 신호가 임계 넘을 때만 자동 개입.

### 적용 방식 모드
- **자동 적용:** 패치를 바로 반영(트레이스로 확인).
- **제안 후 승인:** diff를 보여주고 사용자가 적용/거부(턴 단위 또는 패치 단위).

### 역할 선택
- 어떤 감독 역할을 켤지 체크박스(캐릭터/세계관/플롯/문체/연출).
- 역할별 모델 지정(예: 문체는 소형, 캐릭터는 대형).

### UI 모드 기능 카드 — Supervision Mode Selector
- **목적:** 비용·침습도·속도를 사용자가 직접 조절.
- **사용자 체감 효과:** "그냥 보기"부터 "정밀 교정"까지 슬라이더 하나로.
- **동작 흐름:** 모드 설정 → 파이프라인이 호출 수/적용 정책 결정.
- **호출되는 에이전트:** 모드에 따라 가변.
- **필요한 LLM 호출 수:** Off=0, Audit/Standard=가변, Deep=전체.
- **streaming 사용 시 동작:** Audit-only는 스트림 중 비파괴 표시 가능, 나머지는 완료 후.
- **UI 설정:** 이 카드 자체가 설정 허브.
- **구현 난이도:** 중.
- **MVP / 추후 / 실험:** Off/Audit/Standard = MVP. Deep/제안승인/역할별모델 = 추후.

## 11. 2.5 MVP vs 추후 버전 분리

### 2.5 MVP (꼭 넣을 것)
- 구간 어노테이션 프로토콜(2.1) — 뼈대.
- 보호 마스킹 파이프라인(2.2) — 안전 전제.
- 패치 적용기 + 무결성 게이트(2.3) — 개악 방지.
- Director Arbiter 단일 통합 호출 + 규칙 병합(8).
- 경량 휴리스틱 트리아지 + 조기 종료/예산 캡(9.1, 9.2).
- 핵심 3역할 우선: **캐릭터 / 플롯 / 문체** (RP 불만 빈도 최상위).
- UI: Off / Audit-only / Standard 모드 + 역할 체크박스 + 최소 트레이스.
- fail-safe revert(확신 미달 시 원문 유지).
- streaming: "완료 후 점검" + Audit-only.

### 추후 버전 (1차 이후)
- 세계관/연출 역할 정식 편입(MVP에선 트리아지 신호 + 약식).
- 제안 후 승인(패치 단위 diff 인터랙션).
- 역할별 모델 분리 지정 / Deep 모드.
- 소형 모델 기반 트리아지(휴리스틱 → 학습형).
- 인터랙티브 트레이스(패치 단위 되돌리기, 사유 클릭).
- 사용자 선호 스타일 프로파일(영구 저장 없이 세션 한정 톤 힌트).

### 실험 기능 (검증 필요)
- 스트리밍 중 비파괴 인라인 힌트(완성 전 약식 audit).
- 역할 간 토론 라운드(에이전트가 서로 반박) — 비용/지연 큼, 효과 미검증.
- 자동 강도 적응(최근 패치 채택률 보고 모드 자동 조절).

## 12. Killer Feature 제안

### 12.1 (대표) 연속성 가드 — Continuity Guard
- **목적:** RP에서 가장 큰 몰입 파괴 두 가지 — **유저 행동 무시(agency 침해)** 와 **직전 사건 망각** — 를 전담 탐지하고 **딱 그 구간만** 국소 보정.
- **사용자 체감 효과:** "방금 내가 문 잠갔는데 캐릭터가 그냥 나감", "다쳤다더니 멀쩡함" 같은 김 빠지는 순간이 사라진다. 사용자가 가장 강하게 체감하는 품질 향상.
- **동작 흐름:** 직전 1~2턴 + 유저 입력에서 "확정 사실/행동"을 추출 → 초안이 이를 위반/무시했는지 대조 → 위반 구간만 패치 또는 플래그.
- **호출되는 에이전트:** 플롯/연속성 Steward(주) + 필요 시 캐릭터.
- **필요한 LLM 호출 수:** 1~2 + Director.
- **streaming 사용 시 동작:** 완료 후 점검(가장 가치 높음), Audit 모드 가능.
- **UI 설정:** 민감도(엄격/느슨), 자동 패치 vs 플래그만.
- **구현 난이도:** 중상.
- **MVP / 추후 / 실험:** **MVP 핵심**(플롯 역할의 간판 기능).
- **기존 플러그인과의 차별점:** 단순 교정기는 문장만 본다. 이건 **유저 주도권과 직전 맥락**을 지킨다 — RP 전용 가치.

### 12.2 (보조 후보) 개악 0 보장 — No-Regression Promise
- **목적:** "다듬었더니 더 나빠짐"을 구조적으로 0에 수렴. 패치 전후를 무결성 게이트 + 보수 임계치로 비교해, 개선 확신 없으면 무조건 원문.
- **사용자 체감 효과:** 켜두고 잊어도 손해가 없다 → 항상 켜는 기능이 됨(채택률 = killer 조건).
- **동작 흐름:** 패치 세트 → 게이트/리스크 점수 → 임계 미달 시 revert.
- **호출되는 에이전트:** 없음(또는 경량 검수 1회).
- **필요한 LLM 호출 수:** 0~1.
- **streaming 사용 시 동작:** 무관.
- **UI 설정:** 보수성 강도.
- **구현 난이도:** 중.
- **MVP / 추후 / 실험:** **MVP**.
- **기존 플러그인과의 차별점:** 후처리 플러그인의 최대 불신요소(개악)를 정면 해결.

> 추천 포지셔닝: **Continuity Guard를 마케팅 간판**, **No-Regression Promise를 신뢰 기반**으로 묶어 "안전하게 켜두는 RP 연속성 감독기"로 출시.

### 권장 파이프라인 한눈 요약
```
초안(메인 모델)
  → [마스킹] 보호구간 치환
  → [트리아지] 신호 없으면 원문 그대로 종료
  → [선택된 Steward들] 자기 관할만 어노테이트(구간 단위)

```
모든 단계에서 **유저 입력 불변 / 전체 재작성 금지 / 실패 시 원문 유지** 원칙 유지.
