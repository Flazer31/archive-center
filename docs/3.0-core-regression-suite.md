# Archive Center 3.0 Core Regression Suite

## Purpose

This suite freezes the high-risk behavior that must remain unchanged while
JavaScript calculations move to the backend. It does not connect to a real
MariaDB, ChromaDB, or RisuAI user session.

## Isolation

- JavaScript input/output behavior runs functions extracted directly from
  `Archive Center.js` in a short-lived Node process.
- Backend persistence and rollback behavior uses `httptest`, fake Store, and
  fake Vector implementations.
- No API key, user chat text, runtime data directory, or real session ID is
  included in the fixtures.
- Migration tests use in-memory recording stores and do not copy real rows.

The machine-readable contract is `testdata/core-regression-suite.json`.

## Covered behavior

1. Only RisuAI `model` requests enter main persistence.
2. User-role prompt additions and context-caching-shaped payload tails do not
   disable the main turn engine.
3. Selection-generated substantive user input remains injectable.
4. GigaTrans stores the canonical original and malformed tags fail open.
5. Generic postprocessor final tags and ReKo comparison output select one final
   canonical assistant value.
6. Replayed complete-turn and postprocessor pairs do not duplicate raw or
   derived rows.
7. Embedding failure keeps raw and derived MariaDB results while reporting the
   vector warning.
8. Rollback deletes only from the verified boundary and clamps attached-session
   deletion to `min_from_turn`.
9. Session migration blocks non-empty targets, permits an empty target, copies
   only after preview, and locks migrated source reads/writes.

## Run

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File ".\scripts\test-core-regressions.ps1"
```

POSIX:

```sh
sh ./scripts/test-core-regressions.sh
```

This focused suite supplements, rather than replaces, `go test ./... -count=1`.
