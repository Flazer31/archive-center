# Explicit history trim guard fix

## Symptom

Deleting a normal RisuAI turn left the corresponding raw and derived Archive Center rows intact. The dashboard reported that the shorter visible history might be `/cut` and preserved the DB.

## Cause

The history-trim guard could infer a suffix-window trim without observing a slash command. It then recorded that inference in the same five-minute cache used for actual `/cut` or `/del` input, causing the inference to reinforce itself.

## Fix

- Only the input hook's observed `/cut` or `/del` command creates explicit trim intent.
- Inferred or ambiguous history shapes cannot become command intent.
- Snapshot, persisted-ledger, and ambiguous trim guards require explicit command intent.
- Ordinary verified turn deletion continues to the existing backend rollback decision and cascade cleanup path.

This change narrows an existing protection rule. It does not add another deletion detector or a second rollback implementation.
