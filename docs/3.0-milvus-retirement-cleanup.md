# 3.0 Milvus Retirement Cleanup

## Decision

Archive Center uses ChromaDB as its vector engine. The unfinished Milvus and
Milvus Lite experiment was never selected by the product server, was not
included in release packages, and did not own stored product data.

## Removed

- Milvus configuration fields and environment variables
- Milvus vector implementations and SDK smoke command
- Milvus-only probes, reports, tests, and shadow contracts
- Milvus SDK and transitive Go module dependencies
- Packaging and startup options that suggested Milvus was supported
- Obsolete 2.0 adoption reports that depended on the removed Milvus lane

## Preserved

- MariaDB authority and migrations
- ChromaDB retrieval, vector storage, and embedding configuration
- Generic external vector mode contracts used by ChromaDB
- Release artifact guards for `milvus.db` and `milvus_data`

The artifact guards remain only to prevent obsolete runtime data from being
accidentally shipped. They are not product support for Milvus.

## Data Safety

This cleanup does not migrate or delete user databases. Existing MariaDB and
ChromaDB data paths are unchanged. No release directory or `.runtime` data is
part of the cleanup.

## Verification Gate

- `go test ./... -count=1`
- `scripts/test-core-regressions.ps1`
- PowerShell parser checks for modified packaging/start scripts
- shell syntax checks for modified POSIX scripts
- repository search showing no Milvus runtime/config/module references
